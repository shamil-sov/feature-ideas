# Feature idea: Feature Requests

## Problem

BandLab users often have ideas about what should be improved, added, or changed in the product.

Today these ideas are scattered across social media, comments, support requests, Reddit posts, Discord discussions, and direct feedback.

This makes it hard to understand what users actually want most.

A user might have a strong idea, but there is no obvious place in BandLab where they can share it, see if others agree, and build support around it.

For the product team, this means valuable feedback can be missed or hard to prioritize because it is not collected in one structured place.

## Basic idea

Create a dedicated Feature Requests page where users can submit product ideas and vote on requests from other users.

The goal is to help BandLab understand what people want from the product.

Instead of only collecting feedback passively, BandLab can make feature requests visible, searchable, and measurable.

Users can:

- Share a feature request
- Vote for requests they want
- Comment or add extra context
- Follow ideas they care about
- See whether an idea is being reviewed, planned, or shipped

## How it works

BandLab has a dedicated Feature Requests page.

A user can open the page and browse existing feature requests.

They can filter ideas by category, for example:

- Studio
- Collaboration
- Discovery
- Profile
- Monetization
- AI tools
- Mobile app
- Community

They can also sort ideas by:

- Most voted
- Newest
- Trending
- Recently updated

## Submitting a request

When a user submits a feature request, they provide:

- Title
- Description
- Category
- Optional screenshot or example

Before creating a new request, BandLab can suggest similar existing requests.

This helps reduce duplicates and encourages users to vote or comment on an existing request instead of creating the same request again.

## Voting

Other users can vote for feature requests they want.

Voting should be simple, for example:

👍 I want this

The vote count makes demand visible.

A popular request becomes easier to notice because many users have clearly shown interest in it.

Users can also comment to explain why the idea matters to them.

This is important because vote count tells us how many people want something, but comments explain the actual user need behind it.

## Request statuses

Feature requests can have simple statuses so users know what is happening.

Examples:

- New
- Under review
- Planned
- In progress
- Shipped
- Not planned

This makes the page feel alive and gives users confidence that their feedback is being seen.

When a request status changes, users who voted or followed the request can receive a notification.

## Why users would participate

Users participate because they feel they can influence the product.

They get:

- A clear place to share feature requests
- Public support from other users
- A way to follow progress
- Recognition when their idea gets attention
- A stronger feeling that BandLab listens

For example:

> "I requested this feature and 800 people voted for it."

## Why this is valuable for BandLab

This gives BandLab a structured signal for product demand.

It can help teams understand:

- Which problems are most common
- Which features users request most often
- Which user groups care about specific ideas
- Which requests are gaining momentum
- What language users use to describe their needs

This does not replace product strategy or analytics, but it adds a useful public feedback layer.

Analytics can show what users do.

Feature Requests can show what users wish they could do.

## Moderation and quality

The page needs basic moderation so it does not become noisy.

Possible controls:

- Duplicate detection
- Report request
- Merge duplicate requests
- Hide low-quality or abusive submissions
- Require a minimum description length
- Limit how often a user can submit new ideas

This keeps the board useful and easier to review.

## Internal view

BandLab could also have an internal view for product teams.

The internal view could show:

- Top requests by votes
- Fastest-growing requests
- Requests by category
- Requests by user segment
- Common keywords
- Status changes
- Related comments

This helps turn public feedback into product insight.

## Expected value

This could help with:

- Better understanding of user demand
- More transparent product feedback
- Higher community engagement
- Better prioritization signals
- Fewer scattered feature requests
- More trust between users and BandLab

## Monetization

Free

This feature should probably be free because the main value comes from collecting broad user feedback.
